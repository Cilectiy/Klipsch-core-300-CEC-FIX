# TV → Klipsch Wake Watcher

Скрипт автоматически включает саундбар Klipsch при включении телевизора TCL.

## Как это работает

Скрипт подключается к телевизору по сети и проверяет его реальное состояние через протокол Google Cast (порт `8009`).

В отличие от простой проверки доступности устройства по сети, используется параметр `isStandBy`, который позволяет точно определить, включен телевизор или находится в режиме ожидания.

### Логика работы

- Пока телевизор выключен, скрипт проверяет его состояние **раз в секунду**.
- Как только телевизор переходит из режима ожидания во включенное состояние, скрипт **один раз** отправляет HTTP-запрос на саундбар Klipsch для его пробуждения.
- Пока телевизор остается включенным, дополнительные запросы на саундбар не отправляются.
- Во время работы телевизора проверка его состояния выполняется **раз в 5 секунд**, что уменьшает нагрузку на роутер и сеть.
- После выключения телевизора скрипт снова начинает проверять состояние раз в секунду, ожидая следующего включения.

## Почему используется порт 8009

У телевизоров TCL порт `8008` может быть доступен даже в режиме ожидания, поэтому по нему нельзя определить, действительно ли телевизор включен.

Порт `8009` использует протокол Google Cast и возвращает параметр `isStandBy`, который позволяет корректно определить текущее состояние телевизора.

## Режимы работы

### Обычный режим

По умолчанию скрипт выводит только важные события:

- включение телевизора;
- отправку команды на пробуждение саундбара;
- выключение телевизора.

### Режим отладки

Для просмотра подробной информации можно использовать параметр:

```bash
python3 klipsch.py --debug
```

В этом режиме отображается процесс проверки состояния телевизора и дополнительная диагностическая информация.

## Настраиваемые параметры

```bash
python3 klipsch.py \
    --tv 192.168.31.38 \
    --soundbar-ip 192.168.31.64
```

Параметры:

- `--tv` — IP-адрес телевизора.
- `--soundbar-ip` — IP-адрес саундбара.
- `--debug` — включает подробный вывод для диагностики.

## Особенности

- Не использует HDMI-CEC.
- Не требует ADB.
- Не использует облачные сервисы.
- Работает полностью внутри локальной сети.
- Отправляет команду на включение саундбара только один раз при каждом включении телевизора.
- Не реагирует на доступность телевизора по сети, а использует его фактическое состояние.
