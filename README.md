# TV → Klipsch Wake Watcher

Скрипт автоматически включает саундбар Klipsch при включении телевизора TCL.

## Как это работает

Скрипт подключается к телевизору по сети и проверяет его реальное состояние через протокол Google Cast (порт `8009`).

В отличие от простой проверки доступности устройства по сети, используется параметр `isStandBy`, который позволяет точно определить, включен телевизор или находится в режиме ожидания.

### Логика работы

- Пока телевизор выключен, скрипт проверяет его состояние **раз в секунду**.
- Как только телевизор переходит из режима ожидания во включенное состояние, скрипт **один раз** отправляет HTTP-запрос на саундбар Klipsch для его пробуждения.
- Пока телевизор остается включенным, дополнительные запросы на саундбар не отправляются.
- Во время работы телевизора проверка его состояния выполняется **раз в 5 секунд**, что уменьшает нагрузку на роутер и сеть.
- После выключения телевизора скрипт снова начинает проверять состояние раз в секунду, ожидая следующего включения.

## Почему используется порт 8009

У телевизоров TCL порт `8008` может быть доступен даже в режиме ожидания, поэтому по нему нельзя определить, действительно ли телевизор включен.

Порт `8009` использует протокол Google Cast и возвращает параметр `isStandBy`, который позволяет корректно определить текущее состояние телевизора.

## Режимы работы

### Обычный режим

По умолчанию скрипт выводит только важные события:

- включение телевизора;
- отправку команды на пробуждение саундбара;
- выключение телевизора.

### Режим отладки

Для просмотра подробной информации можно использовать параметр:

```bash
python3 klipsch.py --debug
```

В этом режиме отображается процесс проверки состояния телевизора и дополнительная диагностическая информация.

## Проверка зависимостей

Перед первым запуском рекомендуется убедиться, что Python и все необходимые модули установлены корректно.

### Проверка версии Python

```bash
python3 --version
```

Должна отображаться версия Python **3.13** или новее.

### Проверка необходимых модулей

```bash
python3 -c "import argparse, json, socket, ssl, sys, time"
```

Если команда ничего не вывела и вернулась к приглашению командной строки — все необходимые модули доступны.

### Проверка OpenSSL

```bash
python3 -c "import ssl; print(ssl.OPENSSL_VERSION)"
```

Например:

```text
OpenSSL 3.5.2 5 Aug 2025
```

### Если возникают ошибки вида

```text
ModuleNotFoundError: No module named '_ssl'
```

или

```text
ImportError: libcrypto.so.3 ...
```

значит Python и OpenSSL имеют несовместимые версии.

Для Entware рекомендуется полностью обновить пакеты Python и OpenSSL:

```bash
opkg update
opkg upgrade python3-base
opkg upgrade python3-light
opkg upgrade python3-openssl
opkg upgrade libopenssl
```

После обновления рекомендуется проверить:

```bash
python3 --version
python3 -c "import ssl; print(ssl.OPENSSL_VERSION)"
```

Если обе команды выполняются без ошибок, система готова к запуску скрипта.

## Настраиваемые параметры

```bash
python3 klipsch.py \
    --tv 192.168.31.38 \
    --soundbar-ip 192.168.31.64
```

Параметры:

- `--tv` — IP-адрес телевизора.
- `--soundbar-ip` — IP-адрес саундбара.
- `--debug` — включает подробный вывод для диагностики.

## Изменение IP-адресов

По умолчанию в скрипте используются следующие адреса:

```python
parser.add_argument("--tv", default="192.168.31.38")
parser.add_argument("--soundbar-ip", default="192.168.31.64")
```

Если вы хотите использовать другие устройства по умолчанию, достаточно изменить значения параметра `default` на нужные IP-адреса.

Например:

```python
parser.add_argument("--tv", default="192.168.1.100")
parser.add_argument("--soundbar-ip", default="192.168.1.150")
```

После этого скрипт можно запускать без каких-либо дополнительных параметров:

```bash
python3 klipsch.py
```

Либо IP-адреса можно указывать при каждом запуске:

```bash
python3 klipsch.py \
    --tv 192.168.1.100 \
    --soundbar-ip 192.168.1.150
```

Параметры командной строки имеют приоритет над значениями `default`.

## Автоматический запуск через Entware

Если используется Entware, скрипт можно запускать автоматически при загрузке роутера.

Для этого скопируйте файл `S99klipsch` со страницы и поместите по пути:

```text
/opt/etc/init.d/
```

Замените путь `/opt/root/klipsch.py` до исполняемого .py файла в `S99klipsch`.

```bash
chmod +x /opt/etc/init.d/S99klipsch
```

Основные команды:

Запустить:

```bash
/opt/etc/init.d/S99klipsch start
```

Остановить:

```bash
/opt/etc/init.d/S99klipsch stop
```

Перезапустить:

```bash
/opt/etc/init.d/S99klipsch restart
```

После создания сервиса и перезагрузки роутера Entware автоматически запустит скрипт в фоновом режиме.

## Особенности

- Не использует HDMI-CEC.
- Не требует ADB.
- Не использует облачные сервисы.
- Работает полностью внутри локальной сети.
- Отправляет команду на включение саундбара только один раз при каждом включении телевизора.
- Не реагирует на доступность телевизора по сети, а использует его фактическое состояние.
